# تقرير تصميم الموقع والواجهات
## Premium Bikes Managed Marketplace

**التاريخ:** 2024  
**الإصدار:** 1.0

---

## نظرة عامة

هذا التقرير يوثق منهجية تصميم واجهات المستخدم وتطوير الموقع لمتجر الدراجات الهوائية المميزة. يعتمد المشروع على نهج تصميم نظيف وحديث مع التركيز على تجربة مستخدم ممتازة.

---

## 1. التقنيات والأدوات المستخدمة

### 1.1 إطار العمل الأساسي
- **Laravel 11**: إطار عمل PHP للواجهة الخلفية
- **Blade Templates**: محرك القوالب الخاص بـ Laravel
- **Tailwind CSS 4.0**: إطار عمل CSS للتصميم
- **Vite**: أداة البناء والتجميع

### 1.2 نهج التصميم
- **Mobile-First**: التصميم يبدأ من الشاشات الصغيرة ثم يتوسع
- **Component-Based**: استخدام مكونات قابلة لإعادة الاستخدام
- **Responsive Design**: تصميم متجاوب لجميع الأجهزة
- **Accessibility**: متوافق مع معايير WCAG 2.1 AA

## 2. نظام التصميم (Design System)

### 2.1 نظام الألوان

#### الألوان الأساسية
- **الأزرق الأساسي** (`#1E40AF` - blue-800): للأزرار الرئيسية والروابط
- **الأزرق الفاتح** (`#3B82F6` - blue-500): لحالات التمرير والأزرار الثانوية
- **الأزرق الداكن** (`#1E3A8A` - blue-900): للعناصر العميقة

#### الألوان الثانوية
- **الأخضر** (`#10B981` - emerald-500): لشعارات الشهادات وحالات النجاح
- **البرتقالي** (`#F59E0B` - amber-500): لإبراز برنامج المقايضة والتحذيرات
- **الأحمر** (`#EF4444` - red-500): للأخطاء والإجراءات المدمرة

#### الألوان المحايدة
- مقياس رمادي من `gray-50` إلى `gray-900` للخلفيات والنصوص

### 2.2 نظام الخطوط

#### الخطوط المستخدمة
- **Inter**: الخط الأساسي للنصوص والعناصر (أوزان: 400, 500, 600, 700)
- **Playfair Display**: خط اختياري للعناوين الرئيسية (أوزان: 400, 700)

#### مقياس الأحجام
- **H1**: `text-4xl` (36px) - للعناوين الرئيسية
- **H2**: `text-3xl` (30px) - لعناوين الأقسام
- **H3**: `text-2xl` (24px) - للعناوين الفرعية
- **Body**: `text-base` (16px) - للنص الأساسي
- **Small**: `text-sm` (14px) - للنصوص الثانوية

### 2.3 نظام المسافات

يعتمد على وحدة أساسية 4px:
- `space-1` = 4px
- `space-2` = 8px
- `space-4` = 16px (الوحدة الأساسية)
- `space-6` = 24px
- `space-8` = 32px
- `space-12` = 48px

#### قواعد المسافات
- **البطاقات**: `p-6` (24px) للبطانة الداخلية
- **النماذج**: `mb-4` (16px) بين الحقول
- **الأزرار**: `px-6 py-3` (24px أفقي، 12px عمودي)
- **الأقسام**: `py-12` (48px) بين الأقسام

## 3. المكونات القابلة لإعادة الاستخدام

### 3.1 المكونات الأساسية

#### Layout Components
- **main-layout**: التخطيط الرئيسي مع Header و Footer
- **dashboard-layout**: تخطيط لوحة التحكم مع Sidebar
- **admin-layout**: تخطيط لوحة الإدارة
- **auth-layout**: تخطيط صفحات المصادقة

#### UI Components
- **button**: أزرار بأشكال مختلفة (primary, secondary, danger, ghost)
- **card**: بطاقات للعرض مع تأثيرات hover
- **badge**: شارات للحالات (success, warning, error, info)
- **form-input**: حقول إدخال موحدة
- **product-card**: بطاقة عرض المنتج
- **image-gallery**: معرض الصور
- **filter-panel**: لوحة الفلترة

### 3.2 هيكل المكونات

```blade
{{-- مثال على استخدام المكونات --}}
<x-main-layout title="الصفحة الرئيسية">
    <x-product-card :product="$product" />
    <x-button variant="primary">إضافة للسلة</x-button>
</x-main-layout>
```

## 4. هيكل الواجهات الرئيسية

### 4.1 الصفحة الرئيسية (Homepage)
- **Header**: ثابت في الأعلى مع شعار، بحث، سلة، ملف شخصي
- **Hero Section**: قسم رئيسي مع دعوة للعمل
- **الفئات المميزة**: شبكة 3 أعمدة (متجاوبة)
- **المنتجات المميزة**: عرض متحرك (carousel)
- **قسم "لماذا نحن"**: 3 أعمدة تعرض المميزات

### 4.2 صفحة قائمة المنتجات
- **Sidebar**: فلاتر جانبية (السعر، الماركة، الحالة، السنة)
- **شبكة المنتجات**: 4 أعمدة على سطح المكتب، 2 على الموبايل
- **خيارات الترتيب**: حسب السعر، التاريخ، الدرجة
- **التصفح**: ترقيم الصفحات

### 4.3 صفحة تفاصيل المنتج
- **معرض الصور**: صورة رئيسية مع صور مصغرة
- **معلومات المنتج**: العنوان، السعر، الحالة
- **شهادة الجودة**: عرض بارز إذا كان المنتج معتمد
- **المواصفات**: جدول تفصيلي
- **معلومات البائع**: اسم البائع والتقييم
- **المنتجات ذات الصلة**: عرض متحرك

### 4.4 صفحات الدفع (Checkout)
عملية متعددة الخطوات:
1. **مراجعة السلة**: عرض المنتجات مع ملخص الطلب
2. **معلومات الشحن**: عنوان الشحن وطريقة الشحن
3. **الدفع**: معلومات الدفع والفاتورة
4. **تأكيد الطلب**: رسالة نجاح مع تفاصيل الطلب

### 4.5 لوحة تحكم البائع
- **Sidebar**: قائمة جانبية للتنقل
- **نظرة عامة**: إحصائيات سريعة (منتجات نشطة، إيرادات)
- **قائمة المنتجات**: عرض المنتجات مع إمكانية التعديل
- **الفحوصات المعلقة**: حالة الفحوصات المجدولة

### 4.6 لوحة تحكم الورشة
- **جدول اليوم**: الفحوصات المجدولة
- **الطلبات المعلقة**: طلبات فحص جديدة
- **نموذج الفحص**: قائمة فحص مع إدخال الدرجة

### 4.7 لوحة الإدارة
- **نظرة عامة**: إحصائيات النظام
- **إدارة المستخدمين**: جداول بيانات قابلة للترتيب والفلترة
- **سجل الأنشطة**: تتبع جميع الإجراءات

## 5. مبادئ التصميم

### 5.1 التصميم النظيف والحد الأدنى
- **استخدام المساحات البيضاء**: مساحات واسعة للتنفس
- **البساطة**: تجنب العناصر الزخرفية غير الضرورية
- **التركيز**: هيكل بصري واضح يوجه انتباه المستخدم

### 5.2 الجمالية المميزة
- **جودة الصور**: صور عالية الدقة
- **الانتقالات السلسة**: تأثيرات hover و transitions
- **التفاصيل المصقولة**: عناصر مصممة بعناية

### 5.3 الاتساق
- **مكونات قابلة لإعادة الاستخدام**: استخدام موحد عبر الموقع
- **نظام المسافات**: مسافات متسقة
- **نظام الألوان**: استخدام موحد للألوان

### 5.4 إمكانية الوصول
- **التباين**: نسب تباين ≥ 4.5:1
- **التنقل بالكيبورد**: دعم كامل
- **قارئات الشاشة**: متوافق مع برامج قراءة الشاشة
- **مؤشرات التركيز**: مرئية وواضحة

### 5.5 التصميم المتجاوب
- **Mobile-First**: التصميم يبدأ من الموبايل
- **Breakpoints**:
  - Mobile: < 640px
  - Tablet: 640px - 1024px
  - Desktop: > 1024px
- **أهداف اللمس**: حجم أدنى 44px للأزرار

## 6. معالجة الصور

### 6.1 قواعد الصور
- **نسبة العرض**: 16:9 أو 4:3 للمنتجات
- **الدقة الدنيا**: 1200px عرض
- **التنسيق**: WebP (مفضل)، JPEG، PNG
- **حجم الملف**: حد أقصى 500KB لكل صورة
- **الخلفية**: بيضاء أو محايدة

### 6.2 معالجة الصور
- **تحسين الصور**: تحسين قبل الرفع
- **الصور المتجاوبة**: استخدام `srcset`
- **التحميل البطيء**: lazy loading للصور غير المرئية
- **نص بديل**: مطلوب لإمكانية الوصول

### 7.1 تدفقات المستخدم

### 7.2 التغذية الراجعة
- **حالات النجاح**: رسائل نجاح واضحة
- **حالات الخطأ**: رسائل خطأ مفيدة
- **مؤشرات التحميل**: أثناء العمليات
- **حالات التمرير**: تأثيرات hover واضحة

### 7.3 الكشف التدريجي
- **عرض المعلومات تدريجياً**: إظهار المعلومات حسب الحاجة
- **الأقسام القابلة للطي**: للمعلومات التفصيلية
- **النماذج متعددة الخطوات**: تقسيم النماذج المعقدة

## 8. التكامل مع Domain-Driven Design

### 8.1 محاذاة DDD
- **Bounded Contexts**: كل وحدة لها أنماط واجهة مميزة
- **لغة المجال**: استخدام مصطلحات المجال في الواجهة
- **عرض المجاميع**: عرض جذور المجاميع بشكل بارز
- **مؤشرات الحالة**: تمثيل بصري لحالات الكيانات

### 8.2 الأنماط المستخدمة
- **Strategy Pattern**: لطرق الدفع والشحن
- **State Pattern**: لحالات الطلبات والفحوصات
- **Repository Pattern**: لفصل منطق الوصول للبيانات
